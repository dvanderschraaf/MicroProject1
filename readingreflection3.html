Reading and Response: Schneiderman "The Eight Golden Rules of Interface Design" - 04/13/26

1. Abscent Rule

The rule that feels the most absent from ym micro project is "Strive for Consistecy". While the internal system pf the game is structured and rile based, the interface surrounding it is not consistently organized. For example, instructions arent the cleares nor are they consistent. The lack of consistency makes the experience feel more fragmented than it should.

2. Improved Rule

The rule that would improve my sketch the most is probably "Offer Informative Feedback". My system does provide feedback through movement, color, and score, the interface does not always clearly explain what the feedback means. For example the energy system is there, but its not clear what state it is in, how to menouver it or why it changes. Also the incraese in difficulty as you gain more points is most likely felt but not clearly explained. Adding clear and persistend feedback would make my project so much stronger.

3. Hardest Rule

The hardest rule to implement to my sketch is "Reduce Short-Term Memory Load". This is because, the player is memorizing all the different things at once and if I were to make that visible, it could be overwhelming and make the interface difficult. If I display too much information, it becomes cluttered and distracting. If I display too little, the player has to do all the work to learn. This balance I feel is the hardest.

4. Contradicted Rule

One rule that contradicts my system is "Permit Easy Reversal of Actions". Once you fail in my game, you have to restart. There is no steps back. In a sense. The game ends immidietly and you have to restart. This is intentional but it goes against the idea of letting the users reverse their mistakes.
